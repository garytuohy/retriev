Get started with Retriev in under 5 minutes. Connect Stripe, configure webhooks, and start recovering failed payments.

Quick Start

Retriev integrates with your Stripe account to automatically detect failed payments and initiate AI-powered recovery sequences. Here's how to get up and running.

Create your account

Sign up at retriev.pages.dev/signup. You'll get a 14-day free trial with no credit card required.

Connect Stripe

Click "Connect with Stripe" in onboarding. You'll be redirected to Stripe to authorize read/write access to payment events.

Add webhook endpoint

In Stripe Dashboard → Developers → Webhooks, add our endpoint. This enables real-time failed payment detection.

Watch it work

Once connected, Retriev automatically monitors for failed payments and starts recovery sequences. First recovery typically happens within 48 hours.

⚡ Zero code required

The entire setup is done via OAuth and webhooks. No SDKs to install, no backend changes needed.

Connect Stripe

Retriev uses Stripe Connect for secure, read-write access to your payment events. We never store your secret key.

What permissions we request

  • payment_intent:read — Read payment details
  • payment_intent:write — Retry failed charges
  • customer:read — Fetch customer email/name
  • invoice:read — Read subscription invoices

Disconnecting

You can disconnect Retriev anytime from your Stripe Dashboard → Connected Apps. All recovery sequences will stop immediately.

Webhook Setup

To receive real-time payment events, add our webhook endpoint to your Stripe dashboard.

Endpoint URL

https://retriev-api.garytuohy.workers.dev/api/webhook

Events to enable

In the Stripe webhook configuration, subscribe to these events:

payment_intent.payment_failed
invoice.payment_failed
invoice.paid
customer.subscription.deleted
checkout.session.completed

Verifying the webhook

After adding the endpoint, Stripe will send a test event. Check your Retriev dashboard — you should see "Webhook active" in Settings.

⚠️ Webhook Secret

Stripe provides a webhook signing secret. This is automatically configured when you connect via OAuth — no manual entry needed.

API Authentication

All API requests require authentication via Bearer token in the Authorization header.

Getting your API key

API access is available on the Scale plan. Generate keys from Settings → Integrations.

Using the API key

curl -H "Authorization: Bearer retr_live_xxx" \
  https://retriev-api.garytuohy.workers.dev/api/me

API Endpoints

GET /api/auth/me

Returns the authenticated user's profile.

{
  "user": {
    "id": "usr_abc123",
    "email": "jane@company.com",
    "plan": "growth",
    "stripeConnected": true
  }
}

GET /api/dashboard/stats

Returns recovery metrics for the authenticated account.

{
  "recoveredMonth": 12480,
  "recoveryRate": 73,
  "activeSequences": 24,
  "totalRecovered": 89200

Returns a paginated list of failed payments being tracked.

Email Templates

Retriev uses AI-personalized email sequences. You can customize templates from Settings.

Default sequence

  • Day 1 — Soft reminder, friendly tone
  • Day 3 — Follow-up, still friendly
  • Day 7 — Urgency increases
  • Day 14 — Final notice before account action

Template variables

Use these variables in your templates:

{{customer_name}}  — Customer's first name
{{amount}}         — Payment amount (formatted: $29.00)
{{company_name}}   — Your company name
{{update_link}}    — Stripe-hosted update URL

Dunning Sequences

When a payment fails, Retriev creates a recovery sequence optimized for that customer.

How it works

  1. Stripe webhook fires payment_intent.payment_failed
  2. Retriev creates a sequence entry
  3. AI analyzes customer history to determine optimal send time
  4. First email sent at calculated optimal time
  5. Retries scheduled based on payment method type

Smart retry timing

Our AI determines the best retry window based on:

  • Customer's timezone
  • Historical payment patterns
  • Failure reason (card expired vs. insufficient funds)
  • Previous recovery success rate for similar profiles

Analytics

The Analytics dashboard provides deep insights into your recovery performance.

Key metrics

  • Recovery Rate — % of failed payments successfully recovered
  • Revenue Recovered — Total dollar amount recovered
  • Average Recovery Time — Days from failure to successful payment
  • Churn Prevented — Customers retained who would have churned

AI Insights

The Insights panel surfaces actionable recommendations based on your data:

  • Optimal email send times for your customer base
  • At-risk customers showing churn signals
  • Template tone performance (friendly vs. professional)
